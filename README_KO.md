# 🤖 Awesome GitHub Copilot Customizations

GitHub Copilot 경험을 커뮤니티에서 기여한 [사용자 지정 명령어](#-custom-instructions), [프롬프트](#-reusable-prompts), [채팅 모드](#-custom-chat-modes)로 향상시키세요. 팀의 코딩 표준과 프로젝트 요구사항을 따르는 일관된 AI 지원을 받아보세요.

<details>
<summary><strong>🎯 GitHub Copilot 사용자 정의 기능</strong></summary>

GitHub Copilot은 AI 응답을 사용자 정의하고 특정 워크플로우, 팀 지침 및 프로젝트 요구 사항에 맞게 지원을 조정하는 세 가지 주요 방법을 제공합니다:

| **🧩 [사용자 정의 채팅 모드](#-custom-chat-modes)** | **🎯 [재사용 가능한 프롬프트](#-reusable-prompts)** | **📋 [사용자 정의 명령어](#-custom-instructions)** |
| --- | --- | --- |
| 각 요청에 대한 특정 경계 내에서 채팅 동작, 사용 가능한 도구 및 코드베이스 상호 작용 패턴을 정의합니다.<br><br>**장점:**<br>• 상황에 맞는 지원<br>• 도구 구성<br>• 역할별 워크플로우 | 특정 작업을 위한 재사용 가능하고 독립적인 프롬프트를 생성합니다. 선택적 작업별 지침과 함께 *무엇*을 해야 하는지 설명합니다.<br><br>**장점:**<br>• 반복적인 프롬프트 작성 제거<br>• 팀 간 공유 가능<br>• 변수 및 종속성 지원 | 코드 생성, 리뷰 및 커밋 메시지와 같은 작업에 대한 공통 지침을 정의합니다. 작업을 *어떻게* 수행해야 하는지 설명합니다.<br><br>**장점:**<br>• 모든 채팅 요청에 자동 포함<br>• 저장소 전체 일관성<br>• 다양한 구현 옵션 |

> **💡 프로 팁:** 사용자 정의 명령어는 Copilot 채팅에만 영향을 미칩니다(인라인 코드 완성에는 영향을 미치지 않음). 세 가지 사용자 정의 유형을 모두 결합할 수 있습니다 - 일반 지침에는 사용자 정의 명령어를, 특정 작업에는 프롬프트 파일을, 상호 작용 컨텍스트를 제어하기 위해 채팅 모드를 사용하세요.

</details>
